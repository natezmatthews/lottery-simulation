Notes

  1. Rounding: Redemption count is r × outstanding, rounded down to a whole number. Split among redeemers uses s × redeemers, rounded down for two-ticket buyers; the rest are one-ticket buyers. New offers issued from gap headroom use gap ÷ 2, rounded down. When that value is 0 but gap is still positive (typically a single ticket of headroom), the model issues one offer so the run can finish—acceptable remainder slack for integer rounding.
  2. Expiry and the Expired column: An offer issued on day d is valid through day d + validity − 1 (validity consecutive days, including issue day). At the start of each day, offers past that window drop into Expired. With high daily redemption and redeeming soonest-to-expire offers first, many runs never show expirations—the queue clears before offers age out. Lower the redemption slider (or shorten validity) to see nonzero Expired counts.
  3. (a) Ceiling tickets: Computed after today’s redemptions and before new offers: ceiling = cumulative tickets sold + 2 × (outstanding offers remaining). Counting each remaining offer as two tickets is the conservative “worst case” used so new waves do not oversell. Gap = target − ceiling; new offers come from gap (gap ÷ 2 rounded down; see rounding note).
  4. Small pools: If r × n rounded down is 0 while offers remain (e.g., n = 1), redemption is bumped to 1 so the deterministic schedule does not deadlock.
